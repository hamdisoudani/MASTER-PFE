# skill/

> This folder stores **persistent agent skills** — documented patterns, API references, and lessons learned during real sessions.  
> Every time something non-obvious is discovered, it goes here so future sessions can skip the discovery phase entirely.

## Files

| File | What it covers |
|------|----------------|
| [`composio.md`](./composio.md) | How to use Composio as the bridge to GitHub, E2B, and 1,000+ integrations — tool names, response schemas, patterns, pitfalls, and a quick-reference card |

## How to use these skills

When starting a new session on this repo, read the relevant skill file **before** attempting any integration work.  
The files are self-contained — copy-pasteable examples, gotchas, and quick-reference tables included.

## Contributing

If you (or an agent) discovers a new pattern, fixes a gotcha, or adds a new integration:
1. Update the relevant `.md` file in this folder
2. Add a new file if the topic is new
3. Commit with message `docs(skill): <what changed>`
